## Skill 1.3.5 — Serialize & Deserialize

### DynamoDB Data Types
- Scalar: String (S), Number (N), Binary (B), Boolean (BOOL), Null (NULL)
- Document: List (L), Map (M)
- Set: String Set (SS), Number Set (NS), Binary Set (BS)

### S3 Data Formats
- JSON, CSV, Parquet, Avro
- S3 Select: Query trực tiếp trên S3 objects (SQL)

## Skill 1.3.6 — DynamoDB Operations

| Operation | Mô tả | Limit |
|-----------|--------|-------|
| PutItem | Tạo/replace item | 400KB per item |
| GetItem | Lấy 1 item by key | — |
| UpdateItem | Update attributes | — |
| DeleteItem | Xóa 1 item | — |
| BatchWriteItem | Batch write/delete | Max 25 items |
| BatchGetItem | Batch read | Max 100 items |
| TransactWriteItems | ACID transaction | Max 100 items |

## Skill 1.3.7 — Data Lifecycle

### DynamoDB TTL
- Tự động xóa expired items (background process)
- Attribute chứa expiration timestamp (epoch seconds)
- Không tốn WCU, deleted items xuất hiện trong Streams

### S3 Lifecycle Rules
- Transition: Standard → IA → Glacier → Deep Archive
- Expiration: Tự động xóa sau X ngày

## Skill 1.3.8 — Caching Services

### So sánh

| | ElastiCache Redis | ElastiCache Memcached | DynamoDB DAX |
|---|---|---|---|
| Latency | Sub-millisecond | Sub-millisecond | Microsecond |
| Persistence | ✅ | ❌ | N/A (cache only) |
| Replication | ✅ | ❌ | ✅ |
| Data types | Complex (sorted sets, lists) | Simple key-value | DynamoDB compatible |
| Use case | Session store, leaderboards | Simple caching | DynamoDB acceleration |

### Caching Strategies
- **Lazy Loading**: Cache on read miss (stale data possible)
- **Write-Through**: Cache on every write (always fresh, write penalty)
- **TTL**: Expire cached data after time period
