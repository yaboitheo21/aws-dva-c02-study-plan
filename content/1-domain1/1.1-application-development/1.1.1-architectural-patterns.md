## Architectural Patterns

### Event-Driven
- Producers phát events, consumers xử lý events
- Loosely coupled, scalable
- AWS: EventBridge, SNS, SQS, Lambda triggers

### Microservices
- Ứng dụng chia thành nhiều services nhỏ, độc lập
- Mỗi service có database riêng, deploy riêng
- Giao tiếp qua API (sync) hoặc messaging (async)

### Monolithic
- Toàn bộ ứng dụng trong 1 codebase, 1 deployment unit
- Đơn giản ban đầu, khó scale từng phần

### Choreography vs Orchestration

| | Choreography | Orchestration |
|---|---|---|
| Điều phối | Decentralized — mỗi service tự biết phải làm gì | Central coordinator điều phối workflow |
| Ví dụ | SNS → multiple SQS queues → multiple Lambda | AWS Step Functions |
| Ưu điểm | Loosely coupled, dễ thêm consumer | Dễ theo dõi, error handling tập trung |
| Nhược điểm | Khó debug, khó theo dõi flow | Single point of failure, tightly coupled |

### Fan-out Pattern
- 1 message → nhiều consumers
- SNS topic → multiple SQS queues
- EventBridge → multiple targets
