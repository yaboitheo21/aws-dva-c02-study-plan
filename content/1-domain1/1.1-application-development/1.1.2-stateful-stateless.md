## Stateful vs Stateless

### Stateless
- Không lưu trạng thái giữa các requests
- Mỗi request chứa đủ thông tin để xử lý
- Ví dụ: Lambda functions, API Gateway
- Dễ scale horizontally

### Stateful
- Lưu trạng thái giữa các requests
- Ví dụ: WebSocket connections, session-based apps
- Cần sticky sessions hoặc external state store

### Chuyển Stateful → Stateless

| Vấn đề | Giải pháp |
|---------|-----------|
| Session data | Lưu vào ElastiCache Redis |
| User state | Lưu vào DynamoDB |
| File uploads | Lưu vào S3 |
| Sticky sessions | Dùng ELB sticky sessions (không khuyến khích) |

{{% notice tip %}}
**Exam Tip:** AWS khuyến khích stateless architecture. Khi đề bài hỏi về session management → ElastiCache Redis hoặc DynamoDB.
{{% /notice %}}
