## Tightly Coupled vs Loosely Coupled

### Tightly Coupled
- Components gọi trực tiếp nhau (synchronous)
- Nếu 1 component fail → cả hệ thống fail
- Ví dụ: Direct API calls giữa services

### Loosely Coupled
- Components giao tiếp qua message queue hoặc event bus
- Failure isolation — 1 component fail không ảnh hưởng ngay
- AWS: SQS, SNS, EventBridge, Step Functions

### So sánh

| | Tightly Coupled | Loosely Coupled |
|---|---|---|
| Giao tiếp | Direct calls | Message queue / Event bus |
| Failure | Cascade failures | Failure isolation |
| Scaling | Scale cả hệ thống | Scale từng component |
| Latency | Thấp hơn | Cao hơn (async) |
| Complexity | Đơn giản | Phức tạp hơn |

{{% notice tip %}}
**Exam Tip:** Khi đề bài nói "decouple components" → SQS giữa producer và consumer. Khi nói "fan-out" → SNS + SQS.
{{% /notice %}}
