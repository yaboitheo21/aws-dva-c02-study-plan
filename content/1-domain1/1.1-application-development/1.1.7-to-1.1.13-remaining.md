## Skill 1.1.8 — Messaging Services

### SQS

| Feature | Standard | FIFO |
|---------|----------|------|
| Throughput | Unlimited | 300 msg/s (3000 batching) |
| Delivery | At-least-once | Exactly-once |
| Ordering | Best-effort | Guaranteed |
| Deduplication | No | Yes (5-min window) |

- Visibility Timeout: Default 30s, max 12h
- Dead-Letter Queue: Sau `maxReceiveCount` lần fail
- Long Polling: `WaitTimeSeconds` (1-20s) — giảm empty responses
- Message size: Max 256KB

### SNS
- Pub/Sub model: Topics → Subscriptions (SQS, Lambda, HTTP, Email, SMS)
- Fan-out: SNS → multiple SQS queues
- Message filtering: Filter policies trên subscriptions
- FIFO topics: Ordered, deduplicated

## Skill 1.1.9 — AWS SDKs & APIs

### Credential Resolution Order
1. Code (hardcoded) — **KHÔNG khuyến khích**
2. Environment variables (`AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`)
3. Shared credentials file (`~/.aws/credentials`)
4. EC2 Instance Profile / ECS Task Role / Lambda Execution Role

### AWS CLI
- `--page-size`: Số items per API call
- `--max-items`: Tổng số items trả về
- `--starting-token`: Pagination token

## Skill 1.1.10 — Streaming Data

### Kinesis Data Streams vs Kinesis Data Firehose

| | Data Streams | Data Firehose |
|---|---|---|
| Latency | Real-time (~200ms) | Near real-time (60s buffer) |
| Scaling | Manual (shards) | Auto-scaling |
| Consumers | Custom (Lambda, KCL) | S3, Redshift, OpenSearch, HTTP |
| Retention | 24h-365 days | No retention |
