## Lambda Deployment Packaging

| Method | Size Limit | Use Case |
|--------|-----------|----------|
| ZIP (direct) | 50MB zipped, 250MB unzipped | Small functions |
| ZIP (via S3) | 50MB zipped, 250MB unzipped | Larger packages |
| Container Image | 10GB | Complex dependencies, Docker workflows |
| Layers | 250MB total (5 layers) | Shared libraries |

## SAM Project Structure
```
my-app/
├── template.yaml          # SAM template
├── src/handlers/           # Lambda handlers
├── tests/                  # Unit tests
├── events/                 # Test events
└── samconfig.toml          # SAM CLI config
```

## Elastic Beanstalk Structure
```
my-app/
├── .ebextensions/          # Configuration files (YAML/JSON)
├── .platform/hooks/        # Platform hooks
├── Procfile                # Process definitions
└── app.py                  # Application code
```

## AWS AppConfig
- Feature flags và configuration management
- Gradual deployment of config changes
- Validation: JSON Schema hoặc Lambda validator
- Rollback on errors
- Components: Application → Environment → Configuration Profile

{{% notice tip %}}
**Exam Tip:** AppConfig vs Parameter Store: AppConfig cho feature flags + gradual rollout + validation. Parameter Store cho simple key-value config.
{{% /notice %}}
